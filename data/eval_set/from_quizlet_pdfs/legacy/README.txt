Quizlet PDF → eval_set format
==============================

Note: the live `from_quizlet_pdfs/` tree lives under `DATA/eval_set/` (repo root
`eval_set/` is a symlink). `data/` and `DATA/` are the same folder on a typical
macOS case-insensitive volume.

Generated by:
  scripts/quizlet_pdfs_to_eval_txt.py

Outputs in this folder:
  questions.txt   — same block layout as eval_set/from_youtube_video/questions.txt (N. stem, A–D, blank line)
  answers.txt     — one line per question: "N L" (letter A–D)
  sources.csv     — source PDF absolute paths

INS QAA.pdf (flashcards, options numbered 1–4) is parsed locally.

INS QA.pdf and CA INS QA.pdf (Quizlet Test layout: “k of M”, four choices, “Don’t know”)
are also parsed locally by scripts/quizlet_pdfs_to_eval_txt.py.

Optional OpenAI reconstruction (only if you add PDFs that are not QAA or Test layout):

  export OPENAI_API_KEY=...
  .venv/bin/python3 scripts/quizlet_pdfs_to_eval_txt.py \
    --pdfs "/path/to/other.pdf" \
    --out-dir eval_set/from_quizlet_pdfs \
    --openai-model gpt-4.1 \
    --openai-max-chunks 20 \
    --sleep-ms 150

Review answers: INS QAA keys come from the duplicate “answer” line in the PDF.
Quizlet Test PDFs do not print A–D keys in the PDF text. To still populate answers,
we ship cached Quizlet **flashcard** dumps under `data/` and a matcher script:

  .venv/bin/python3 scripts/fill_answers_from_quizlet_flashcards.py \\
    --questions eval_set/from_quizlet_pdfs/questions.txt \\
    --out eval_set/from_quizlet_pdfs/answers.txt

That aligns each stem to the same Quizlet set’s **definition** (correct phrase) and
picks the closest A–D option. Remaining `?` rows need manual review or
`--answers-merge-from`. See `answer_key_notice.txt`.
